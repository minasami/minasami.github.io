# Website Usefulness Standard

Purpose: define what useful means for this digital health platform.

## Useful means
- A visitor knows where to start within seconds.
- Each audience has a clear next step.
- The site teaches, guides and proves.
- Important claims are easy to verify.
- Reviewers can move from story to evidence quickly.
- Recruiters can understand role fit quickly.
- Healthcare leaders can learn a practical operating model.
- Partners can see what is ready and what still needs proof.

## Audience needs

### Executive
Needs a fast path to the leadership model, readiness lens and decision logic.

### Recruiter
Needs a fast path to role fit, project evidence, technical communication and judgment.

### Partner
Needs a fast path to the case study, proof boundary, reliability and security roadmap.

### Learner
Needs a fast path to the learning lab, readiness assessment and implementation playbook.

## Usefulness rule
Every major page should answer three questions:

1. What is this?
2. Why does it matter?
3. What should I do next?

## Next upgrade
Add or improve visible paths that help every audience reach the right proof or action in one click.
