# Case Study Standard

Every case study should make real work easier to evaluate.

## Required structure

Each major case study should answer:

1. What problem existed?
2. What constraints shaped the work?
3. What was Mina's role?
4. What system or workflow was created?
5. What evidence supports the story?
6. What should not be overstated?
7. What should a serious reviewer ask next?

## Useful evidence

Evidence can include live systems, screenshots, workflow maps, SOPs, dashboards, role definitions, counts, reports, governance artifacts, training material, and before/after process descriptions.

## Claim boundaries

Avoid overstating budget ownership, credential status, clinical authority, enterprise software certification, or P&L ownership.

## Priority case studies

- Medicine Support Hub
- Employee Support Program

## Reviewer route

Case study page → Case Study Proof System → Proof Room → Claims Registry → Executive Conversation Kit

## Decision rule

If a case study cannot show problem, action, system, evidence and boundaries, improve it before making stronger claims elsewhere.
