Checklist

Before evidence goes public

1. Redaction

Remove names, emails, phone numbers, private notes, personal identifiers and any unnecessary admin-only details.

2. Demo data

Use sample beneficiaries, sample organizations and sample records. Never publish real private operational data.

3. Proof label

Each screenshot or video should state what it proves: landing page, dashboard, workflow, role model or reporting surface.

4. Claim boundary

Each proof item should also state what it does not prove yet, such as adoption, certification or clinical production use.

5. Review route

Link evidence back to the Proof Trail, Technical Proof Checkpoint, Reviewer Checklist and Claims Registry.

6. Update rule

When stronger evidence arrives, upgrade the proof page first and only then upgrade public wording.
