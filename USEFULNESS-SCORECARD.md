# Platform Usefulness Scorecard

Last reviewed: 2026-07-08

## Purpose

Measure whether the platform is becoming more useful, not merely larger. This scorecard keeps improvement work focused on practical visitor outcomes.

## Core question

Can a serious visitor quickly understand the work, choose the right path, verify proof and take the next useful action?

## Scorecard

| Dimension | What good looks like | Evidence to review |
| --- | --- | --- |
| Clarity | A visitor understands the platform purpose within seconds. | Homepage, Command Center, Decision Navigator. |
| Direction | Each audience has an obvious next step. | Architecture map, navigation, calls to action. |
| Decision support | Visitors can move from unclear decision to structured brief and readiness view. | Decision Navigator, Decision Brief, Readiness Assessment. |
| Proof | Strong claims are easy to verify. | Proof Room, Claims Registry, Medicine Support Hub. |
| Trust | Claims stay evidence-backed and bounded. | Accuracy notes, proof links, review guides. |
| Conversion | Serious visitors can contact, review or continue without confusion. | Contact route, recruiter resources, executive routes. |
| Maintainability | New changes do not create duplicate routes, duplicate scripts or fragile flows. | Architecture map, shared asset inventory, quality gates. |

## Monthly review questions

1. Which page helped visitors reach proof fastest?
2. Which path created the clearest next action?
3. Which claims need stronger public evidence?
4. Which pages or scripts are duplicating work?
5. Which route should be simplified before adding anything new?

## Usefulness signals

Review these together:

- Decision Brief generation.
- Readiness report print or save actions.
- Proof Room and Medicine Support Hub visits.
- Contact clicks after proof or assessment routes.
- Recruiter and executive route usage.

## Improvement rule

A change is useful when it improves clarity, decision quality, proof, trust or action. If it only adds more content without improving one of those areas, it should wait.
