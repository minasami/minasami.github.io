# Demo Evidence Standard

Purpose: keep Medicine Support Hub demo evidence public-safe, useful and honest.

## Demo data only
Use sample names, sample organizations and sample records. Do not show private beneficiary, medical, contact or admin-sensitive details.

## Recommended flow
1. Problem
2. Public platform surface
3. Admin dashboard structure
4. Request-to-impact workflow
5. Proof pages
6. Claim boundary

## Good evidence shows
- What the platform looks like
- How the workflow is structured
- Which role or dashboard concept is being demonstrated
- Where reviewers can inspect proof

## Good evidence does not claim
- Clinical production use
- External adoption
- Certification
- National scale
- Enterprise validation

Evidence first. Stronger wording second.
