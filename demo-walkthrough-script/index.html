Recording script

Use this sequence

0-10 sec: Open with the problem

Medicine support is hard to govern when requests, beneficiaries, medicines, approvals and impact records are scattered.

10-25 sec: Show the public platform surface

Show the landing page and explain the promise: one operating platform from request to measurable impact.

25-45 sec: Show the admin dashboard

Use demo data to show organizations, programs, beneficiaries and timeline records.

45-65 sec: Show the workflow model

Explain how a request can move through review, approval, fulfillment, tracking and impact visibility.

65-80 sec: Show the proof stack

Point reviewers to the evidence gallery, technical checkpoint, proof trail and claims registry.

80-90 sec: Close with the boundary

State clearly that the demo proves product surfaces and workflow thinking, not clinical production use or external adoption.

Important rule

Use demo data only.

Do not show real beneficiary details, phone numbers, emails, medical notes, internal admin details or private records. The demo should prove capability without exposing people.
